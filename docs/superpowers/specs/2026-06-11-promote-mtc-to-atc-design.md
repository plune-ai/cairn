# Promote MTC → ATC — Design

- **Date:** 2026-06-11
- **Status:** Approved (brainstorming)
- **Related code:** `src/agent/index.ts` (`runAutomate`, `runDesign` numbering), `src/artifacts/testcase-md.ts`
  (`parseTestCaseMd` / `renderTestCaseMd`), `src/browser/` (gateway, locator verify), `src/tui/` (RunDetailScreen).

## Context / Problem

`design` writes test cases as **ATC** (auto) and **MTC** (manual) markdown in `runs/<id>/testcases/`. `automate`
generates `@playwright/test` only for ATC, filtering MTC out (`src/agent/index.ts:539`):

```ts
const cases = allCases.filter((c) => c.execution !== "manual" && !c.id.startsWith("MTC"));
```

After review, a user often decides a specific MTC is actually automatable and wants it as ATC. By hand this means:
edit `execution`, renumber `id` to the next free ATC slot (so it doesn't collide with existing `ATC-…-001…`),
rename the file, and re-collect selectors if missing. Error-prone — hence a **promote** feature.

## Decisions (from brainstorming)

1. **In-place** — rename `MTC-…md` → `ATC-…md`; the original MTC disappears.
2. **Decoupled** — promote only converts the `.md`; code is generated by a subsequent `automate` run.
3. **Selectors hybrid** — if the `.md` has no selectors: fill from `runs/<id>/study.json` (offline); any
   still-missing refs → open the page (URL from `report.json` + session) and verify live. No session → warn, leave empty.
4. **Traceability** — add `Promoted from MTC-<oldId>` to the case's `## Traceability`.
5. **CLI + TUI**, sharing one `promoteCase` function.

## Architecture

New isolated module **`src/promote/`** (single responsibility), consumed by both the CLI command and the TUI action.
It depends only on `fs`, `testcase-md` parse/render helpers, and (for the live fallback) an injected selector-collector
so the core stays browser-free and unit-testable.

### `promoteCase(runDir, caseId, deps?)` — convert-only

```ts
interface PromoteDeps {
  /** Live browser fallback for refs missing from study.json. Omit → offline-only (warn on miss). */
  collectLive?: (url: string, refs: string[]) => Promise<Map<string, string>>;
}
interface PromoteResult {
  oldId: string;          // MTC-SUITE-001
  newId: string;          // ATC-SUITE-004
  oldFile: string;
  newFile: string;
  selectorsFilled: number;
  missingRefs: string[];  // refs still without a locator
  warning?: string;       // e.g. "3 refs have no selector; generated code will be incomplete"
}
async function promoteCase(runDir: string, caseId: string, deps?: PromoteDeps): Promise<PromoteResult>;
```

Steps:
1. Resolve `testcases/<caseId>.md`; require `caseId` to start with `MTC` (else: not a manual case → error).
2. Parse suite from `MTC-<SUITE>-NNN`; scan `testcases/` for existing `ATC-<SUITE>-*`; `newNum = max + 1`
   (zero-padded to 3). `newId = ATC-<SUITE>-<newNum>`.
3. If the parsed `## Selectors` is empty → recover the case's `elementRefs` from `report.json` by matching the
   case **title** (the `.md` doesn't store elementRefs, and its id `MTC-…` differs from report's `tc-…` id), then
   call `collectSelectors` (below).
4. Rewrite frontmatter: `id: <newId>`, `execution: auto`, `status: ❌ Not implemented`,
   `automation: tests/ui/<suite-lower>/<newId-lower>.spec.ts`. Append/extend `## Traceability` with
   `| Promoted from | <oldId> |`.
5. Rename the file MTC→ATC (in-place). Return `PromoteResult`.

### `collectSelectors(runDir, elementRefs, deps?)` — hybrid

```ts
async function collectSelectors(
  runDir: string,
  elementRefs: string[],
  deps?: PromoteDeps,
): Promise<{ selectors: { label: string; locator: string }[]; missing: string[] }>;
```

1. Read `study.json` → map `ref → element`; build `getByRole(...)` locators from role/name (same shape `design`
   uses in `renderTestCaseMd`). Fill every ref found.
2. For refs absent from study **and** when `deps.collectLive` is provided → read URL from `report.json`, call
   `collectLive(url, missingRefs)`, merge results. Refs still missing are returned in `missing`.

### CLI — `lex-bot promote`

```
lex-bot promote --run <dir> --cases MTC-SUITE-001,MTC-SUITE-003 [--session <name>] [--session-file <path>]
```

- For each id: build `collectLive` from `makeGateway` + the session (only if a session is given), call `promoteCase`.
- Print `MTC-… → ATC-…` per case, plus any `warning`. `--session` is used **only** for the live selector fallback;
  offline promotes need none.

### TUI — action on the Cases tab

- In `RunDetailScreen`, on a case whose id starts with `MTC`, key **`a`** = "promote to ATC".
- Calls `promoteCase` (offline `collectLive` omitted unless the run's session is known to the TUI — v1: offline-only
  in TUI, show the warning + hint to use the CLI with `--session` for the live fallback).
- On success, reload artifacts so the case appears as its new `ATC-…` id.

## Numbering (the core correctness point)

- Suite = the middle segment of the id (`MTC-<SUITE>-NNN`).
- Next ATC number = `max(existing ATC-<SUITE>-NNN) + 1`, scanned from `testcases/` filenames/frontmatter.
- Guarantees no collision with existing ATC ids. Promoting several MTCs in one CLI call processes them sequentially
  so each sees the previous one's new number.

## Error handling

- `caseId` not found / not `MTC*` → throw a clear error (CLI prints it; TUI shows it inline).
- `report.json` / `study.json` unreadable → selectors stay empty, `warning` set, promote still succeeds (decoupled —
  the user can re-collect later or run `automate` and see the gap at validate).
- Live fallback without a session → skipped with a warning, not an error.

## Testing

- `promoteCase`: fixture `testcases/` (+ `report.json`, `study.json`) → asserts new ATC number = max+1, file renamed,
  frontmatter (`id`/`execution`/`automation`) updated, traceability appended, selectors filled from study.
- `collectSelectors`: offline branch (study has refs) and the missing-ref path (with a mocked `collectLive`).
- CLI: argument parsing + multi-case sequential numbering (mock gateway).
- TUI: `RunDetailScreen` render — `a` on an MTC case invokes promote (mock `promoteCase`) and reloads.

## Out of scope (YAGNI)

- Generating the spec during promote (that's `automate`, deliberately decoupled).
- Demoting ATC → MTC.
- A bulk "promote all MTC" shortcut (explicit ids only, to keep it intentional).
